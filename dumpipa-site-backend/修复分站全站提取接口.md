# 修复分站全站提取接口

## 问题分析

**问题**：访问 `https://www.dumpipa.com/api/tasks` 返回 `{"ok":0,"msg":"未提供认证令牌"}`

**原因分析**：
1. 分站后端（`dumpipa-site-backend`）的 `getAllTasks` 会调用主站API
2. 分站后端会获取分站token并传递给主站API
3. 如果主站API（`dumpipa-backend`）还在使用 `authenticate` 中间件，会返回"未提供认证令牌"错误
4. 或者分站token无效，但主站API尝试验证失败

## 解决方案

### 方案 1：修改分站后端，允许不传递token（已修改）

**已修改**：`src/controllers/taskController.ts` 和 `src/services/proxyService.ts`

- 如果 `getSiteToken()` 返回 `null`，传递 `undefined` 给 `proxyRequest`
- `createProxyClient` 支持 `undefined`，不设置 Authorization header
- 这样主站API会以未登录方式访问

### 方案 2：确保主站API使用 optionalAuth（需要部署）

**主站后端**（`dumpipa-backend`）需要：
1. `src/routes/tasks.ts` 中 `/api/tasks` 使用 `optionalAuth` ✅（已修改）
2. `src/controllers/taskController.ts` 的 `getAllTasks` 支持未登录用户 ✅（已修改）
3. **在主站服务器上部署并重启服务**

## 当前状态

### 分站后端（dumpipa-site-backend）
- ✅ `router.get('/', taskController.getAllTasks)` - 公开路由，无中间件
- ✅ `getAllTasks` 支持 token 不存在的情况
- ✅ `proxyRequest` 支持不传递 token

### 主站后端（dumpipa-backend）
- ✅ `router.get('/', optionalAuth, taskController.getAllTasks)` - 使用 optionalAuth
- ✅ `getAllTasks` 支持未登录用户访问

## 临时解决方案

如果无法立即更新主站代码，可以修改分站后端，在调用主站API时，如果token无效，不传递token：

```typescript
// 在 getAllTasks 中
const siteToken = await getSiteToken()
// 如果token不存在或无效，传递 undefined
const result = await proxyRequest(
  'GET',
  '/api/tasks',
  { page, page_size: pageSize, include_done: includeDone },
  siteToken || undefined
)
```

**已实现**：分站后端已经支持这个逻辑。

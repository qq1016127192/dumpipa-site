# 修复主站后端全站提取接口

## 问题分析

**问题**：访问 `https://www.dumpipa.com/api/tasks` 返回 `{"ok":0,"msg":"未提供认证令牌"}`

**原因**：
1. 主站后端（`dumpipa-site-backend`）的 `getAllTasks` 函数会通过 `proxyRequest` 调用主站API
2. 主站后端在调用主站API时，会传递分站token
3. 如果主站API（`dumpipa-backend`）还在使用 `authenticate` 中间件，会返回"未提供认证令牌"错误
4. 主站服务器上的代码可能还没有更新

## 解决方案

### 方案 1：更新主站服务器代码（推荐）

在主站服务器上更新 `dumpipa-backend` 的代码：
1. 将 `src/routes/tasks.ts` 中的 `authenticate` 改为 `optionalAuth`
2. 修改 `src/controllers/taskController.ts` 中的 `getAllTasks` 函数，支持未登录用户

### 方案 2：修改主站后端，允许未登录访问

修改主站后端（`dumpipa-site-backend`）的 `getAllTasks` 函数：
- 如果分站token不存在或无效，不传递token，让主站API以未登录方式访问

### 方案 3：主站后端直接查询数据库（不推荐）

主站后端直接查询数据库，不通过主站API。但这会破坏架构设计。

## 推荐方案

**推荐使用方案 1**：在主站服务器上更新代码。

### 需要修改的文件

1. **`dumpipa-backend/src/routes/tasks.ts`**：
   ```typescript
   router.get('/', optionalAuth, taskController.getAllTasks)
   ```

2. **`dumpipa-backend/src/controllers/taskController.ts`**：
   - 支持 `req.user` 可能为 `undefined`
   - 未登录用户返回全站任务

### 部署步骤

1. 在主站服务器上，进入 `dumpipa-backend` 目录
2. 拉取最新代码或手动修改文件
3. 重启后端服务

## 临时解决方案

如果无法立即更新主站代码，可以修改主站后端，在token无效时允许未登录访问：

```typescript
// 在 getAllTasks 函数中
const siteToken = await getSiteToken()
// 如果token不存在，不传递token，让主站API以未登录方式访问
const result = await proxyRequest(
  'GET',
  '/api/tasks',
  { page, page_size: pageSize, include_done: includeDone },
  siteToken || undefined // 如果token不存在，传递undefined
)
```

但是，这需要主站API支持未登录访问（使用 `optionalAuth`）。
